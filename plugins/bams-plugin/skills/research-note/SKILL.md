# 소프트웨어 연구노트 작성 가이드

## 개요

연구노트는 연구의 기획, 계획, 과정, 결과, 성과 등 전주기적 과정을 기록한 1차 기록물이자 원자료입니다.
제3자가 연구를 이해하고 재현할 수 있도록 상세하게 작성해야 합니다.

## 연구노트 필수 요소

1. **과제명**: 연구/개발 과제의 공식 명칭
2. **과제번호**: 과제 관리 번호
3. **과제기간**: 과제 수행 기간 (YYYY-MM-DD ~ YYYY-MM-DD)
4. **작성일**: 연구노트 작성 일자 (YYYY-MM-DD)
5. **책임자**: 과제 책임자 성명 (한글)

## 전산 분야 연구노트 주요 기록 항목

### 1. 아이디어 설명
- 연구/개발 목적과 배경
- 해결하고자 하는 문제
- 접근 방식 및 예상 결과

### 2. Flow Chart 및 Module 설명
- 시스템 아키텍처 다이어그램
- 모듈별 역할과 기능 설명
- 데이터 흐름 및 의존성

### 3. 일자별 소스코드 기록
- 구현된 코드 및 변경사항
- Git 커밋 링크 또는 해시
- 주요 함수/클래스 설명

### 4. Debugging 및 Error 기록
- 발견된 에러 내용
- 원인 분석
- 해결 방안 및 적용 결과

### 5. 실험 조건 기록
- 개발/테스트 환경 (OS, 런타임 버전 등)
- 하드웨어 사양
- 외부 의존성 및 버전

### 6. 공동 연구 기록
- 주고받은 메시지/메일 요약
- 협업 내용 및 역할 분담

### 7. 회의 내용
- 회의 일시, 참석자
- 논의 내용 및 결정사항
- Action Items

## 연구노트 작성 워크플로우

### 필수: Git 커밋 로그 참조

**연구노트 작성 전 반드시 Git 커밋 로그와 커밋 메시지를 확인해야 합니다.**

```bash
# 최근 커밋 로그 확인 (상세)
git log --oneline -20

# 특정 기간 커밋 확인
git log --since="2024-01-01" --until="2024-01-31" --oneline

# 커밋별 변경 파일 확인
git log --stat -10

# 특정 커밋의 상세 내용
git show <commit-hash>
```

커밋 로그에서 추출할 정보:
- 작업 일자 및 시간
- 변경된 파일 목록
- 커밋 메시지 (작업 내용 요약)
- 작업자 정보

### 신규 연구노트 생성

1. **Git 커밋 로그 수집**: `git log` 명령으로 기간별 커밋 내역 확인
2. 프로젝트 정보 확인: 연구명, 과제번호, 책임자 정보
3. 현재 프로젝트 상태 파악: 주요 파일 구조
4. `references/template.md` 템플릿을 기반으로 작성
5. 작성 완료 후 저장 위치: 프로젝트 `docs/research-notes/` 폴더

### 기존 연구노트 업데이트

1. **Git 커밋 로그 확인**: 마지막 연구노트 작성일 이후 커밋 확인
2. 기존 연구노트 파일 확인
3. 새로운 날짜 섹션 추가
4. 커밋 메시지 기반으로 변경사항 기록

## 작성 원칙

1. **즉시성**: 작업 수행 직후 바로 기록
2. **상세성**: 제3자가 재현 가능하도록 구체적으로 작성
3. **객관성**: 사실에 기반하여 가중 없이 기록
4. **완전성**: 실패한 실험도 반드시 기록 (원인 분석 포함)
5. **템플릿 준수**: `references/template.md` 템플릿에 정의된 항목만 작성. 저장소 URL 등 템플릿에 없는 항목을 임의로 추가하지 않음

## 파일 저장

- 파일명 형식: `YYYY-MM-DD_주제.md` 또는 `연구노트_과제명_YYYYMMDD.md`
- 저장 위치: 프로젝트 내 `docs/research-notes/` 폴더
