# Best Practice: data-binding-mapper

- **트리거**: design-import 파이프라인의 Phase C에서 design-director가 위임. F3 ui-diff-applier와 병렬 실행 가능. F9 ssr-csr-decider와도 병렬.
- **입력**:
  - `.crew/artifacts/design/{slug}/guide-recomposition/placeholder-slots.json` (F2 산출물)
  - `.crew/artifacts/design/{slug}/guide-recomposition/normalized-guide.json`
  - (선택) `.crew/artifacts/design/{slug}/ssr-decision/rendering-strategy.json` (F9 산출물)
  - (선택) `src/app/api/{endpoint}/route.ts` — 데이터 스키마 참조 (Read-only)
  - (선택) `prisma/schema.prisma` — 데이터 모델 참조 (Read-only)
- **출력**:

- **모델**: gpt-5-codex (codex CLI 위임) + sonnet controller

## 2. 자주 발생하는 실수 3건

### 실수 1: 실시간 데이터 슬롯을 server로 잘못 마킹
- **증상**: binding-map.json에서 `realtimePrice`, `stockTicker`, `liveCount` 류 슬롯이 `"component_type": "server"`로 마킹됨. frontend-engineering이 RSC fetch 패턴으로 구현 후 실시간 업데이트 불가.
- **원인**: 슬롯명 분석 없이 일괄 server 마킹. F9 rendering-strategy.json을 참조하지 않음.
- **회피**: 슬롯명에 `realtime`, `live`, `stream`, `ws`, `socket` 키워드 감지 시 자동 `"component_type": "client"` 마킹. F9 결과 존재 시 우선 적용. `"schema_status": "unverified"`로 명시.

### 실수 2: API endpoint 미확정 시 TBD 마킹 누락으로 frontend 블로킹
- **증상**: binding-map.json의 `api_path` 필드가 `/api/unknown` 또는 빈 문자열로 저장. frontend-engineering이 구현 불가 상태로 블로킹.
- **원인**: backend-engineering API 스키마 미확인 상태에서 추측 경로 기입.
- **회피**: 불확실 시 반드시 `"api_path": "TBD"`, `"schema_status": "unverified"` 마킹. binding-map.json 최상단에 `"tbd_count": N` 집계 + backend-engineering에 확인 요청 메모 기록.

### 실수 3: fetch-snippets.tsx에 `"use client"` directive 삽입
- **증상**: RSC fetch 스니펫 파일 상단에 `"use client"`가 추가됨. Server Component로 사용할 수 없어 번들 크기 증가.
- **원인**: Client Component 슬롯 스니펫 작성 중 파일 전체에 directive 적용 실수.
- **회피**: fetch-snippets.tsx는 Server Component 스니펫(디폴트)과 Client Component 스니펫(TanStack Query)을 섹션 주석으로 분리. `"use client"` directive는 CC 섹션 내부 코드 예시에만 주석 형태로 명시.

## 3. 권장 패턴

- **패턴 A — F9 우선 적용**: `rendering-strategy.json` 존재 시 각 슬롯의 `component_type`을 F9 결정값으로 초기화 후 슬롯별 오버라이드 판단. 불일치 발생 시 design-director 경유 F9 협의.
- **패턴 B — RSC fetch 디폴트 스니펫 구조**:
  ```tsx
  // Target: src/app/{page}/page.tsx
  // Strategy: RSC fetch (OQ4=c 디폴트) — Generated by data-binding-mapper
  async function get{SlotName}() {
    const res = await fetch('/api/{endpoint}', { cache: 'no-store' });
    if (!res.ok) throw new Error('Failed to fetch');
    return res.json();
  }
  export default async function Page() { const data = await get{SlotName}(); return <>{data.field}</>; }
  ```
- **패턴 C — loading-states.md 3항 필수**: (1) Suspense boundary 위치 (컴포넌트명 + 경로), (2) error boundary 범위, (3) skeleton UI 구조 (`<SkeletonCard>` 등). frontend-engineering 핸드오프 ready 형식.
